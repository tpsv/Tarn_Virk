End-to-End Cybersecurity Lab: Building and Defending a Secure Virtual Environment

Overview

In an era where threat actors continually evolve their techniques, I designed and implemented a full-scale cybersecurity lab tailored for red teaming, threat detection, and endpoint defense validation. This environment mimics real-world attack scenarios and showcases my ability to both emulate adversaries and proactively defend systems.

Virtual Environment Architecture

Using VMware Workstation Pro, I architected a secure and segmented virtual lab. The environment includes:

  • A Windows 10 VM functioning as the victim workstation.
  • A Kali Linux VM operating as the attacker console.
  • Tight network isolation to simulate internal compromise scenarios.

This modular setup enables controlled attack execution and real-time telemetry capture—critical for both offensive and defensive testing.

Adversary Simulation with Sliver C2

To emulate real-world threats, I deployed Sliver, an open-source Command & Control (C2) framework developed by Bishop Fox.

  • Server Setup: Installed and configured Sliver server on the Linux VM, enabling implant generation and interactive session control.
  • Payload Creation: Generated Windows payloads to simulate attacker presence on the victim machine.
  • Session Management: Established encrypted C2 channels to exfiltrate data and execute post-exploitation tasks such as LSASS memory dumping.

Key Skill Demonstrated: Understanding of post-exploitation techniques, process injection, and privilege escalation.

Endpoint Detection & Telemetry Collection

To monitor and analyze malicious behavior, I integrated LimaCharlie EDR and Sysmon on the Windows VM.

  • LimaCharlie Agent Deployment: Registered the VM as a sensor and ingested rich telemetry including process creation, file access, and network connections.
  • Sysmon Configuration: Used SwiftOnSecurity’s configuration to log granular endpoint activity, such as command-line arguments and parent-child process relationships.

This telemetry fed into LimaCharlie’s timeline and detection engine, providing a foundation for custom detection engineering.

Simulated Attacks: Credential Dumping and Shadow Copy Deletion

Credential Dumping via LSASS:

procdump -n lsass.exe -s lsass.dmp

This mimics attacker attempts to extract user credentials from memory—one of the most critical post-exploitation techniques. LimaCharlie successfully detected this through its SENSITIVE_PROCESS_ACCESS event logs.

Volume Shadow Copy Deletion:

vssadmin delete shadows /all

Using this command, I simulated ransomware tactics aimed at sabotaging recovery mechanisms. A high-fidelity detection rule was crafted to monitor access to vssadmin.exe, alert, and block malicious commands in real-time. LimaCharlie’s deny_tree feature neutralized the threat.

Detection Engineering & Response Automation

One of the most impactful components of the lab was crafting and tuning Detection & Response (D&R) rules within LimaCharlie. These rules dynamically react to suspicious behavior in real-time.

Detection Example:

            event: SENSITIVE_PROCESS_ACCESS
            op: ends with
            path: event/*/TARGET/FILE_PATH
            value: lsass.exe

Response:

            action: report
            name: LSASS access

Later enhanced with blocking logic to neutralize ransomware payloads preemptively.

Fine-Tuning for Accuracy: Reducing False Positives

Not every detection is a threat. I demonstrated my defensive acumen by identifying and reducing false positives through:

  • Crafting suppression rules.
  • Contextual filtering (e.g., isolating svchost.exe executions outside System32).
  • Improving detection granularity without compromising sensitivity.

This reflects my understanding of balancing security and usability—crucial in any enterprise SOC.

Outcome

The environment I built is modular, replicable, and fully suited for continuous testing and blue team practice. It stands as a testament to my capability in offensive and defensive security, and my readiness to operate in high-stakes cybersecurity environments.
